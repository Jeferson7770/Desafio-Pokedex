# Security Policy

## Supported Versions

Security fixes are applied to the main branch first.

For production users, always track the latest stable state from main and apply updates quickly.

## Reporting a Vulnerability

Please do not disclose security vulnerabilities in public issues.

Report privately by email:

- security@suafince.com.br

Include:

- summary of the vulnerability
- affected area and version or commit
- reproduction steps or proof of concept
- impact assessment
- suggested mitigation if available

## Response Targets

Best effort targets:

- initial acknowledgment: up to 3 business days
- triage result: up to 7 business days
- mitigation or fix timeline: shared after triage

## Disclosure Process

1. Report received and acknowledged.
2. Team validates and assesses severity.
3. Fix is prepared and reviewed.
4. Coordinated disclosure is agreed when appropriate.
5. Public advisory may be published after fix availability.

## Safe Harbor

We support good-faith security research.

Please:

- avoid privacy violations
- avoid service disruption
- avoid data exfiltration
- avoid social engineering and physical attacks

## Scope

This policy covers this repository and code paths directly maintained by Fince.

Third-party services and dependencies are handled with best effort and coordinated with vendors when needed.
